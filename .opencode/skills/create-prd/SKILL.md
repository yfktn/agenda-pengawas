# Create PRD

Tujuan:
Membuat Product Requirement Document yang lengkap.

Langkah:

1. Identifikasi masalah yang ingin diselesaikan.
2. Identifikasi pengguna.
3. Buat user stories.
4. Buat functional requirements.
5. Buat non-functional requirements.
6. Buat acceptance criteria.
7. Simpan hasil ke docs/prd.md
8. Tanyakan saya jika ada yang kurang menurut kamu

MANDATORY RULES
1. Jangan membuat struktur custom PRD
2. Gunakan template yang ada di skills/create-prd/templates/prd-template.md
3. siapkan heading sesuai dengan struktur
4. kalau ada yang tidak bisa dikerjakan dengan lengkap, tuliskan TBD
5. Jangan hapus bagian yang ada

Output:
docs/prd.md
